La CLI ClawC démarre par défaut le [TUI](/docs/tui) lorsqu'il est exécuté sans aucun argument.

```bash
clawc
```

Mais elle accepte également les commandes documentées sur cette page. Cela vous permet d'interagir avec ClawC par programme.

```bash
clawc run "Explain how closures work in JavaScript"
```

---

### tui

Démarrez l’interface utilisateur du terminal ClawC.

```bash
clawc [project]
```

#### Options

| Option                                   | Court | Description                                                                   |
| ---------------------------------------- | ----- | ----------------------------------------------------------------------------- |
| <nobr><code>{"--continue"}</code></nobr> | `-c`  | Continuer la dernière session                                                 |
| <nobr><code>{"--session"}</code></nobr>  | `-s`  | ID de session pour continuer                                                  |
| <nobr><code>{"--fork"}</code></nobr>     |       | Forker la session en continuant (à utiliser avec `--continue` ou `--session`) |
| <nobr><code>{"--prompt"}</code></nobr>   |       | Prompt à utiliser                                                             |
| <nobr><code>{"--model"}</code></nobr>    | `-m`  | Modèle à utiliser sous forme de fournisseur/modèle                            |
| <nobr><code>{"--agent"}</code></nobr>    |       | Agent à utiliser                                                              |
| <nobr><code>{"--port"}</code></nobr>     |       | Port d'écoute                                                                 |
| <nobr><code>{"--hostname"}</code></nobr> |       | Nom d'hôte d'écoute                                                           |

## Commandes

La CLI ClawC dispose également des commandes suivantes.

---

### agent

Gérer les agents pour ClawC.

```bash
clawc agent [command]
```

---

### attach

Attachez un terminal à un serveur backend ClawC déjà en cours d'exécution démarré via les commandes `serve` ou `web`.

```bash
clawc attach [url]
```

Cela permet d'utiliser le TUI avec un backend ClawC distant. Par exemple :

```bash
# Start the backend server for web/mobile access
clawc web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
clawc attach http://10.20.30.40:4096
```

#### Options

| Option                                   | Court | Description                                                                                        |
| ---------------------------------------- | ----- | -------------------------------------------------------------------------------------------------- |
| <nobr><code>{"--dir"}</code></nobr>      |       | Répertoire de travail dans lequel démarrer TUI                                                     |
| <nobr><code>{"--continue"}</code></nobr> | `-c`  | Continuer la dernière session                                                                      |
| <nobr><code>{"--session"}</code></nobr>  | `-s`  | ID de session pour continuer                                                                       |
| <nobr><code>{"--fork"}</code></nobr>     |       | Dupliquer la session lors de la reprise (à utiliser avec `--continue` ou `--session`)              |
| <nobr><code>{"--password"}</code></nobr> | `-p`  | Mot de passe d'authentification de base (par défaut `CLAWC_SERVER_PASSWORD`)                    |
| <nobr><code>{"--username"}</code></nobr> | `-u`  | Nom d'utilisateur d'authentification de base (par défaut `CLAWC_SERVER_USERNAME` ou `clawc`) |

#### create

Créez un nouvel agent avec une configuration personnalisée.

```bash

```

Cette commande vous guidera dans la création d'un nouvel agent avec un prompt système personnalisé et une configuration d'outils.

#### list

```bash
clawc agent list
```

### auth

Commande pour gérer les informations d’identification et la connexion des fournisseurs.

```bash
clawc auth [command]
```

---

#### login

ClawC est alimenté par la liste des fournisseurs sur [Models.dev](https://models.dev), vous pouvez donc utiliser `clawc auth login` pour configurer les clés API pour tout fournisseur que vous souhaitez utiliser. Ceci est stocké dans `~/.local/share/clawc/auth.json`.

```bash
clawc auth login
```

Lorsque ClawC démarre, il charge les fournisseurs à partir du fichier d'informations d'identification, ainsi que les clés définies dans vos environnements ou un fichier `.env` dans votre projet.

---

#### list

Liste tous les fournisseurs authentifiés tels qu'ils sont stockés dans le fichier d'informations d'identification.

```bash
clawc auth list
```

Ou la version courte.

```bash
clawc auth ls
```

---

#### logout

Vous déconnecte d'un fournisseur en l'effaçant du fichier d'informations d'identification.

```bash
clawc auth logout
```

### github

Gérez l'agent GitHub pour l'automatisation du dépôt.

```bash
clawc github [command]
```

---

#### install

Installez l'agent GitHub dans votre dépôt.

```bash
clawc github install
```

Cela configure le flux de travail GitHub Actions nécessaire et vous guide tout au long du processus de configuration. [En savoir plus](/docs/github).

---

#### run

Exécutez l'agent GitHub. Ceci est généralement utilisé dans les GitHub Actions.


| <nobr><code>{"--token"}</code></nobr> | Jeton d'accès personnel GitHub                       |

### MCP

Gérer les serveurs Model Context Protocol.

```bash
clawc mcp [command]
```

---

#### add

Ajoutez un serveur MCP à votre configuration.

```bash
clawc mcp add
```

Cette commande vous guidera dans l’ajout d’un serveur MCP local ou distant.

---

#### list

Liste tous les serveurs MCP configurés et leur état de connexion.

```bash
clawc mcp list
```

Ou utilisez la version courte.

```bash
clawc mcp ls
```

---

#### auth

Authentifiez-vous auprès d'un serveur MCP compatible OAuth.

```bash
clawc mcp auth [name]
```

Si vous ne fournissez pas de nom de serveur, vous serez invité à sélectionner parmi les serveurs compatibles OAuth disponibles.

Vous pouvez également répertorier les serveurs compatibles OAuth et leur statut d'authentification.

```bash
clawc mcp auth list
```

Ou utilisez la version courte.

```bash
clawc mcp auth ls
```

---

#### logout

Supprimez les informations d'identification OAuth pour un serveur MCP.

```bash
clawc mcp logout [name]
```

---

#### debug

Déboguer les problèmes de connexion OAuth pour un serveur MCP.

```bash
clawc mcp debug <name>
```

### models

Liste tous les modèles disponibles auprès des fournisseurs configurés.

```bash
clawc models [provider]
```

Cette commande affiche tous les modèles disponibles parmi vos fournisseurs configurés au format `provider/model`.

Ceci est utile pour déterminer le nom exact du modèle à utiliser dans [votre config](/docs/config/).

Vous pouvez éventuellement transmettre un ID de fournisseur pour filtrer les modèles en fonction de ce fournisseur.

```bash
clawc models anthropic
```

#### Options

| Option                                  | Description                                                                                |
| --------------------------------------- | ------------------------------------------------------------------------------------------ |
| <nobr><code>{"--refresh"}</code></nobr> | Actualisez le cache des modèles à partir de models.dev                                     |
| <nobr><code>{"--verbose"}</code></nobr> | Utiliser une sortie de modèle plus détaillée (inclut des métadonnées telles que les coûts) |

Utilisez l'option `--refresh` pour mettre à jour la liste des modèles mis en cache. Ceci est utile lorsque de nouveaux modèles ont été ajoutés à un fournisseur et que vous souhaitez les voir dans ClawC.

```bash
clawc models --refresh
```

### run

Exécutez clawc en mode non interactif en transmettant directement un prompt.

```bash
clawc run [message..]
```

Ceci est utile pour les scripts, l'automatisation ou lorsque vous souhaitez une réponse rapide sans lancer le TUI complet. Par exemple :

```bash "clawc run"
clawc run Explain the use of context in Go
```

Vous pouvez également vous connecter à une instance `clawc serve` en cours d'exécution pour éviter les temps de démarrage à froid du serveur MCP à chaque exécution :

```bash
# Start a headless server in one terminal
clawc serve

# In another terminal, run commands that attach to it
clawc run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Options

| Option                                   | Court | Description                                                                                        |
| ---------------------------------------- | ----- | -------------------------------------------------------------------------------------------------- |
| <nobr><code>{"--command"}</code></nobr>  |       | La commande à exécuter, utilisez le message pour les arguments                                     |
| <nobr><code>{"--continue"}</code></nobr> | `-c`  | Continuer la dernière session                                                                      |
| <nobr><code>{"--session"}</code></nobr>  | `-s`  | ID de session pour continuer                                                                       |
| <nobr><code>{"--fork"}</code></nobr>     |       | Forker la session en continuant (à utiliser avec `--continue` ou `--session`)                      |
| <nobr><code>{"--share"}</code></nobr>    |       | Partager la session                                                                                |
| <nobr><code>{"--model"}</code></nobr>    | `-m`  | Modèle à utiliser sous forme de fournisseur/modèle                                                 |
| <nobr><code>{"--agent"}</code></nobr>    |       | Agent à utiliser                                                                                   |
| <nobr><code>{"--file"}</code></nobr>     | `-f`  | Fichier(s) à joindre au message                                                                    |
| <nobr><code>{"--format"}</code></nobr>   |       | Format : par défaut (formaté) ou json (événements JSON bruts)                                      |
| <nobr><code>{"--title"}</code></nobr>    |       | Titre de la session (utilise un prompt tronqué si aucune valeur n'est fournie)                     |
| <nobr><code>{"--attach"}</code></nobr>   |       | Connectez-vous à un serveur clawc en cours d'exécution (par exemple, http://localhost:4096)     |
| <nobr><code>{"--password"}</code></nobr> | `-p`  | Mot de passe d'authentification de base (par défaut `CLAWC_SERVER_PASSWORD`)                    |
| <nobr><code>{"--username"}</code></nobr> | `-u`  | Nom d'utilisateur d'authentification de base (par défaut `CLAWC_SERVER_USERNAME` ou `clawc`) |
| <nobr><code>{"--dir"}</code></nobr>      |       | Répertoire d'exécution, ou chemin sur le serveur distant lors de l'attachement                     |
| <nobr><code>{"--variant"}</code></nobr>  |       | Variante du modèle (effort de raisonnement spécifique au fournisseur)                              |
| <nobr><code>{"--thinking"}</code></nobr> |       | Afficher les blocs de réflexion                                                                    |
| <nobr><code>{"--port"}</code></nobr>     |       | Port du serveur local (port aléatoire par défaut)                                                  |

### serve

Démarrez un serveur ClawC sans interface graphique pour un accès API. Consultez le [serveur docs](/docs/server) pour l'interface complète HTTP.

```bash
clawc serve
```

Cela démarre un serveur HTTP qui fournit un accès API aux fonctionnalités d'ClawC sans l'interface TUI. Définissez `CLAWC_SERVER_PASSWORD` pour activer l'authentification de base HTTP (le nom d'utilisateur par défaut est `clawc`).

#### Options

| Option                                   | Description                                                |
| ---------------------------------------- | ---------------------------------------------------------- |
| <nobr><code>{"--port"}</code></nobr>     | Port d'écoute                                              |
| <nobr><code>{"--hostname"}</code></nobr> | Nom d'hôte d'écoute                                        |
| <nobr><code>{"--mdns"}</code></nobr>     | Activer la découverte mDNS                                 |
| <nobr><code>{"--cors"}</code></nobr>     | Origines de navigateur supplémentaires pour autoriser CORS |

### stats

Affichez les statistiques d'utilisation et de coût des jetons pour vos sessions ClawC.

```bash
clawc stats
```

#### Options

| Option                                  | Description                                                                                                           |
| --------------------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| <nobr><code>{"--days"}</code></nobr>    | Afficher les statistiques des N derniers jours (depuis le début)                                                      |
| <nobr><code>{"--tools"}</code></nobr>   | Nombre d'outils à afficher (tous)                                                                                     |
| <nobr><code>{"--models"}</code></nobr>  | Afficher la répartition de l'utilisation du modèle (masqué par défaut). Passez un numéro pour afficher les N premiers |
| <nobr><code>{"--project"}</code></nobr> | Filtrer par projet (tous les projets, chaîne vide : projet actuel)                                                    |

### export

Exportez les données de session au format JSON.

```bash
clawc export [sessionID]
```

Si vous ne fournissez pas d'ID de session, vous serez invité à sélectionner parmi les sessions disponibles.

---

### import

Importez les données de session à partir d'un fichier JSON ou d'une URL de partage ClawC.

```bash
clawc import <file>
```

Vous pouvez importer à partir d'un fichier local ou d'une URL de partage ClawC.

```bash
clawc import session.json
clawc import https://clawc.ai/s/abc123
```

### uninstall

Désinstallez ClawC et supprimez tous les fichiers associés.

```bash
clawc uninstall
```

#### Options

| Option                                      | Court | Description                                         |
| ------------------------------------------- | ----- | --------------------------------------------------- |
| <nobr><code>{"--keep-config"}</code></nobr> | `-c`  | Conserver les fichiers de configuration             |
| <nobr><code>{"--keep-data"}</code></nobr>   | `-d`  | Conserver les données de session et les instantanés |
| <nobr><code>{"--dry-run"}</code></nobr>     |       | Afficher ce qui serait supprimé sans supprimer      |
| <nobr><code>{"--force"}</code></nobr>       | `-f`  | Ignorer les invites de confirmation                 |

### upgrade

Met à jour clawc vers la dernière version ou une version spécifique.

```bash
clawc upgrade [target]
```

Pour mettre à niveau vers la dernière version.

```bash
clawc upgrade
```

Pour passer à une version spécifique.

```bash
clawc upgrade v0.1.48
```

#### Options

| Option                                 | Court | Description                                                     |
| -------------------------------------- | ----- | --------------------------------------------------------------- |
| <nobr><code>{"--method"}</code></nobr> | `-m`  | La méthode d'installation utilisée ; curl, npm, pnpm, bun, brew |

## Variables d'environnement

ClawC peut être configuré à l'aide de variables d'environnement.

| Variables                             | Type    | Description                                                                       |
| ------------------------------------- | ------- | --------------------------------------------------------------------------------- |
| `CLAWC_AUTO_SHARE`                 | booléen | Partager automatiquement des sessions                                             |
| `CLAWC_GIT_BASH_PATH`              | chaîne  | Chemin vers l'exécutable Git Bash sur Windows                                     |
| `CLAWC_CONFIG`                     | chaîne  | Chemin d'accès au fichier de configuration                                        |
| `CLAWC_TUI_CONFIG`                 | chaîne  | Chemin d'accès au fichier de configuration TUI                                    |
| `CLAWC_CONFIG_DIR`                 | chaîne  | Chemin d'accès au répertoire de configuration                                     |
| `CLAWC_CONFIG_CONTENT`             | chaîne  | Contenu de configuration JSON en ligne                                            |
| `CLAWC_DISABLE_AUTOUPDATE`         | booléen | Désactiver les vérifications automatiques des mises à jour                        |
| `CLAWC_DISABLE_PRUNE`              | booléen | Désactiver le nettoyage des anciennes données                                     |
| `CLAWC_DISABLE_TERMINAL_TITLE`     | booléen | Désactiver les mises à jour automatiques des titres du terminal                   |
| `CLAWC_PERMISSION`                 | chaîne  | Configuration des autorisations JSON intégrées                                    |
| `CLAWC_DISABLE_DEFAULT_PLUGINS`    | booléen | Désactiver les plugins par défaut                                                 |
| `CLAWC_DISABLE_LSP_DOWNLOAD`       | booléen | Désactiver les téléchargements automatiques du serveur LSP                        |
| `CLAWC_ENABLE_EXPERIMENTAL_MODELS` | booléen | Activer les modèles expérimentaux                                                 |
| `CLAWC_DISABLE_AUTOCOMPACT`        | booléen | Désactiver le compactage automatique du contexte                                  |
| `CLAWC_DISABLE_CLAUDE_CODE`        | booléen | Désactiver la lecture de `.claude` (prompt + compétences)                         |
| `CLAWC_DISABLE_CLAUDE_CODE_PROMPT` | booléen | Désactiver la lecture `~/.claude/CLAUDE.md`                                       |
| `CLAWC_DISABLE_CLAUDE_CODE_SKILLS` | booléen | Désactiver le chargement de `.claude/skills`                                      |
| `CLAWC_DISABLE_MODELS_FETCH`       | booléen | Désactiver la récupération de modèles à partir de sources distantes               |
| `CLAWC_FAKE_VCS`                   | chaîne  | Faux fournisseur VCS à des fins de test                                           |
| `CLAWC_CLIENT`                     | chaîne  | Identifiant du client (par défaut `cli`)                                          |
| `CLAWC_ENABLE_EXA`                 | booléen | Activer les outils de recherche Web Exa                                           |
| `CLAWC_SERVER_PASSWORD`            | chaîne  | Activer l'authentification de base pour `serve`/`web`                             |
| `CLAWC_SERVER_USERNAME`            | chaîne  | Remplacer le nom d'utilisateur d'authentification de base (par défaut `clawc`) |
| `CLAWC_MODELS_URL`                 | chaîne  | URL personnalisé pour récupérer la configuration des modèles                      |
