يبدأ CLI الخاص بـ ClawC افتراضيا واجهة [TUI](/docs/tui) عند تشغيله بدون أي معاملات.

```bash
clawc
```

كما يدعم أيضا أوامر موثقة في هذه الصفحة، ما يتيح لك التفاعل مع ClawC برمجيا.

```bash
clawc run "Explain how closures work in JavaScript"
```

---

### tui

بدء واجهة المستخدم terminal (TUI) الخاصة بـ ClawC.

```bash
clawc [project]
```

#### الخيارات

| الخيار                                   | المختصر | الوصف                                                             |
| ---------------------------------------- | ------- | ----------------------------------------------------------------- |
| <nobr><code>{"--continue"}</code></nobr> | `-c`    | متابعة الجلسة الأخيرة                                             |
| <nobr><code>{"--session"}</code></nobr>  | `-s`    | معرّف الجلسة للمتابعة                                             |
| <nobr><code>{"--fork"}</code></nobr>     |         | تفريع الجلسة عند المتابعة (يستخدم مع `--continue` أو `--session`) |
| <nobr><code>{"--prompt"}</code></nobr>   |         | الموجّه المراد استخدامه                                           |
| <nobr><code>{"--model"}</code></nobr>    | `-m`    | النموذج المراد استخدامه بصيغة provider/model                      |
| <nobr><code>{"--agent"}</code></nobr>    |         | الوكيل المراد استخدامه                                            |
| <nobr><code>{"--port"}</code></nobr>     |         | المنفذ الذي يتم الاستماع عليه                                     |
| <nobr><code>{"--hostname"}</code></nobr> |         | اسم المضيف الذي يتم الاستماع عليه                                 |

### attach

إرفاق terminal بخادم الواجهة الخلفية لـ ClawC قيد التشغيل بالفعل، والذي تم تشغيله عبر الأمرين `serve` أو `web`.

```bash
clawc attach [url]
```

يتيح ذلك استخدام واجهة TUI مع واجهة خلفية لـ ClawC تعمل عن بعد. على سبيل المثال:

```bash
# ابدأ خادم الواجهة الخلفية للوصول عبر الويب/الجوال
clawc web --port 4096 --hostname 0.0.0.0

# في محطة طرفية (terminal) أخرى، اربط TUI بالواجهة الخلفية قيد التشغيل
clawc attach http://10.20.30.40:4096
```

#### الرايات

| الراية                                   | المختصر | الوصف                                                                             |
| ---------------------------------------- | ------- | --------------------------------------------------------------------------------- |
| <nobr><code>{"--dir"}</code></nobr>      |         | دليل العمل الذي ستبدأ منه واجهة TUI                                               |
| <nobr><code>{"--continue"}</code></nobr> | `-c`    | متابعة آخر جلسة                                                                   |
| <nobr><code>{"--session"}</code></nobr>  | `-s`    | معرّف الجلسة للمتابعة                                                             |
| <nobr><code>{"--fork"}</code></nobr>     |         | تفريع الجلسة عند المتابعة (استخدمه مع `--continue` أو `--session`)                |
| <nobr><code>{"--password"}</code></nobr> | `-p`    | كلمة مرور المصادقة الأساسية (الافتراضي `CLAWC_SERVER_PASSWORD`)                |
| <nobr><code>{"--username"}</code></nobr> | `-u`    | اسم مستخدم المصادقة الأساسية (الافتراضي `CLAWC_SERVER_USERNAME` أو `clawc`) |

### auth

أمر لإدارة بيانات الاعتماد وتسجيل الدخول لمزودي الخدمة.

```bash
clawc auth [command]
```

---

#### login

يعتمد ClawC على قائمة المزودين في [Models.dev](https://models.dev)، لذا يمكنك استخدام `clawc auth login` لتهيئة مفاتيح API لأي مزود ترغب باستخدامه. يتم حفظ ذلك في `~/.local/share/clawc/auth.json`.

```bash
clawc auth login
```

عند تشغيل ClawC يقوم بتحميل المزودين من ملف بيانات الاعتماد، وكذلك أي مفاتيح معرّفة في متغيرات البيئة لديك أو في ملف `.env` ضمن مشروعك.

---

#### list

يسرد جميع المزودين الذين تم توثيقهم كما هم محفوظون في ملف بيانات الاعتماد.

```bash
clawc auth list
```

أو النسخة المختصرة.

```bash
clawc auth ls
```

---

#### logout

يسجلك خارج مزود عبر حذفه من ملف بيانات الاعتماد.

```bash
clawc auth logout
```

### models

عرض جميع النماذج المتاحة من المزودين المُهيأين.

```bash
clawc models [provider]
```

يعرض هذا الأمر جميع النماذج المتاحة عبر المزودين المُهيأين لديك بصيغة `provider/model`.

هذا مفيد لتحديد اسم النموذج الدقيق الذي ستستخدمه في [تهيئتك](/docs/config/).

يمكنك اختياريا تمرير معرّف مزود لتصفية النماذج حسب ذلك المزود.

```bash
clawc models anthropic
```

#### الرايات

| الراية                                  | الوصف                                                         |
| --------------------------------------- | ------------------------------------------------------------- |
| <nobr><code>{"--refresh"}</code></nobr> | تحديث ذاكرة التخزين المؤقت للنماذج من models.dev              |
| <nobr><code>{"--verbose"}</code></nobr> | استخدام مخرجات أكثر تفصيلا للنماذج (تشمل بيانات مثل التكاليف) |

استخدم الراية `--refresh` لتحديث قائمة النماذج المخزنة مؤقتا. يفيد ذلك عند إضافة نماذج جديدة إلى مزود وتريد رؤيتها في ClawC.

```bash
clawc models --refresh
```

### run

تشغيل clawc في وضع غير تفاعلي عبر تمرير موجه مباشرة.

```bash
clawc run [message..]
```

هذا مفيد للبرمجة النصية والأتمتة، أو عندما تريد إجابة سريعة دون تشغيل واجهة TUI كاملة. على سبيل المثال.

```bash "clawc run"
clawc run Explain the use of context in Go
```

يمكنك أيضا الإرفاق بمثيل `clawc serve` قيد التشغيل لتجنّب زمن الإقلاع البارد لخوادم MCP في كل تشغيل:

```bash
# ابدأ خادمًا بلا واجهة في محطة طرفية واحدة
clawc serve

# في محطة طرفية أخرى، شغّل الأوامر التي ترتبط به
clawc run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### الرايات

| الراية                                   | المختصر | الوصف                                                                             |
| ---------------------------------------- | ------- | --------------------------------------------------------------------------------- |
| <nobr><code>{"--command"}</code></nobr>  |         | الأمر المراد تشغيله؛ استخدم الرسالة كوسائط                                        |
| <nobr><code>{"--continue"}</code></nobr> | `-c`    | متابعة الجلسة الأخيرة                                                             |
| <nobr><code>{"--session"}</code></nobr>  | `-s`    | معرّف الجلسة للمتابعة                                                             |
| <nobr><code>{"--fork"}</code></nobr>     |         | تفريع الجلسة عند المتابعة (يستخدم مع `--continue` أو `--session`)                 |
| <nobr><code>{"--share"}</code></nobr>    |         | مشاركة الجلسة                                                                     |
| <nobr><code>{"--model"}</code></nobr>    | `-m`    | النموذج المراد استخدامه بصيغة provider/model                                      |
| <nobr><code>{"--agent"}</code></nobr>    |         | الوكيل المراد استخدامه                                                            |
| <nobr><code>{"--file"}</code></nobr>     | `-f`    | ملف/ملفات لإرفاقها بالرسالة                                                       |
| <nobr><code>{"--format"}</code></nobr>   |         | التنسيق: default (منسق) أو json (أحداث JSON خام)                                  |
| <nobr><code>{"--title"}</code></nobr>    |         | عنوان للجلسة (يستخدم موجهًا مقتطعًا إن لم تُحدَّد قيمة)                           |
| <nobr><code>{"--attach"}</code></nobr>   |         | الإرفاق بخادم clawc قيد التشغيل (مثل http://localhost:4096)                    |
| <nobr><code>{"--password"}</code></nobr> | `-p`    | كلمة مرور المصادقة الأساسية (الافتراضي `CLAWC_SERVER_PASSWORD`)                |
| <nobr><code>{"--username"}</code></nobr> | `-u`    | اسم مستخدم المصادقة الأساسية (الافتراضي `CLAWC_SERVER_USERNAME` أو `clawc`) |
| <nobr><code>{"--dir"}</code></nobr>      |         | دليل التشغيل، أو المسار على الخادم البعيد عند الإرفاق                             |
| <nobr><code>{"--variant"}</code></nobr>  |         | متغير النموذج (جهد الاستدلال الخاص بالمزود)                                       |
| <nobr><code>{"--thinking"}</code></nobr> |         | عرض كتل التفكير                                                                   |
| <nobr><code>{"--port"}</code></nobr>     |         | منفذ الخادم المحلي (الافتراضي منفذ عشوائي)                                        |

### serve

بدء خادم ClawC بدون واجهة للوصول عبر API. راجع [توثيق الخادم](/docs/server) لواجهة HTTP الكاملة.

```bash
clawc serve
```

يشغّل هذا خادم HTTP يوفّر وصولا عبر API لوظائف clawc دون واجهة TUI. اضبط `CLAWC_SERVER_PASSWORD` لتفعيل مصادقة HTTP الأساسية (اسم المستخدم الافتراضي `clawc`).

#### الرايات

| الراية                                   | الوصف                                     |
| ---------------------------------------- | ----------------------------------------- |
| <nobr><code>{"--port"}</code></nobr>     | المنفذ الذي يتم الاستماع عليه             |
| <nobr><code>{"--hostname"}</code></nobr> | اسم المضيف الذي يتم الاستماع عليه         |
| <nobr><code>{"--mdns"}</code></nobr>     | تفعيل اكتشاف mDNS                         |
| <nobr><code>{"--cors"}</code></nobr>     | أصول/منشأات إضافية للمتصفح للسماح بـ CORS |

### stats

عرض إحصاءات استخدام الرموز والتكلفة لجلسات ClawC لديك.

```bash
clawc stats
```

#### الرايات

| الراية                                  | الوصف                                                                     |
| --------------------------------------- | ------------------------------------------------------------------------- |
| <nobr><code>{"--days"}</code></nobr>    | عرض الإحصاءات لآخر N يومًا (الافتراضي: كل الوقت)                          |
| <nobr><code>{"--tools"}</code></nobr>   | عدد الأدوات المطلوب عرضها (الافتراضي: الكل)                               |

### upgrade

تحديث clawc إلى أحدث إصدار أو إلى إصدار محدد.

```bash
clawc upgrade [target]
```

للترقية إلى أحدث إصدار.

```bash
clawc upgrade
```

للترقية إلى إصدار محدد.

```bash
clawc upgrade v0.1.48
```

#### الرايات

| الراية                                 | المختصر | الوصف                                                       |
| -------------------------------------- | ------- | ----------------------------------------------------------- |
| <nobr><code>{"--method"}</code></nobr> | `-m`    | طريقة التثبيت المستخدمة: curl أو npm أو pnpm أو bun أو brew |

## متغيرات البيئة

يمكن تهيئة ClawC باستخدام متغيرات البيئة.

| المتغير                               | النوع   | الوصف                                                     |
| ------------------------------------- | ------- | --------------------------------------------------------- |
| `CLAWC_AUTO_SHARE`                 | boolean | مشاركة الجلسات تلقائيا                                    |
| `CLAWC_GIT_BASH_PATH`              | string  | مسار ملف Git Bash التنفيذي على Windows                    |
| `CLAWC_CONFIG`                     | string  | مسار ملف التهيئة                                          |
| `CLAWC_CONFIG_DIR`                 | string  | مسار دليل التهيئة                                         |
| `CLAWC_CONFIG_CONTENT`             | string  | محتوى تهيئة JSON مُضمّن                                   |
| `CLAWC_DISABLE_AUTOUPDATE`         | boolean | تعطيل التحقق التلقائي من التحديثات                        |
| `CLAWC_DISABLE_PRUNE`              | boolean | تعطيل تنقية البيانات القديمة                              |
| `CLAWC_DISABLE_TERMINAL_TITLE`     | boolean | تعطيل تحديث عنوان terminal تلقائيا                        |
| `CLAWC_PERMISSION`                 | string  | تهيئة أذونات JSON مُضمّنة                                 |
| `CLAWC_DISABLE_DEFAULT_PLUGINS`    | boolean | تعطيل الإضافات الافتراضية                                 |
| `CLAWC_DISABLE_LSP_DOWNLOAD`       | boolean | تعطيل تنزيل خوادم LSP تلقائيا                             |
| `CLAWC_ENABLE_EXPERIMENTAL_MODELS` | boolean | تفعيل النماذج التجريبية                                   |
| `CLAWC_DISABLE_AUTOCOMPACT`        | boolean | تعطيل ضغط السياق تلقائيا                                  |
| `CLAWC_DISABLE_CLAUDE_CODE`        | boolean | تعطيل القراءة من `.claude` (الموجه + المهارات)            |
| `CLAWC_DISABLE_CLAUDE_CODE_PROMPT` | boolean | تعطيل قراءة `~/.claude/CLAUDE.md`                         |
| `CLAWC_DISABLE_CLAUDE_CODE_SKILLS` | boolean | تعطيل تحميل `.claude/skills`                              |
| `CLAWC_DISABLE_MODELS_FETCH`       | boolean | تعطيل جلب النماذج من مصادر بعيدة                          |
| `CLAWC_FAKE_VCS`                   | string  | مزود VCS وهمي لأغراض الاختبار                             |
| `CLAWC_CLIENT`                     | string  | معرّف العميل (الافتراضي `cli`)                            |
| `CLAWC_ENABLE_EXA`                 | boolean | تفعيل أدوات بحث الويب من Exa                              |
| `CLAWC_SERVER_PASSWORD`            | string  | تفعيل المصادقة الأساسية لخادمي `serve`/`web`              |
| `CLAWC_SERVER_USERNAME`            | string  | تجاوز اسم مستخدم المصادقة الأساسية (الافتراضي `clawc`) |
| `CLAWC_MODELS_URL`                 | string  | رابط مخصص لجلب تهيئة النماذج                              |
